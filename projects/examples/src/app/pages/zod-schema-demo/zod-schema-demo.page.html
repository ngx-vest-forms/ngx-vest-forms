• Zod schema: passed as the 2nd argument to Vest's create() via Standard Schema V1.
  • Per-field rules: test() callbacks using enforce handle field-level validation.
  • Full runs: suite.run(model) executes both the Zod schema and all test() callbacks.
  • Focused runs: suite.only(field).run(model) runs only the matching test() — the Zod schema is skipped by design.
  • ngx-vest-forms uses focused runs for per-field validation, so the Zod schema acts as a structural safety net on full validation.
