Form approach
@if (approach() === 'split-wrappers') {
  • Each field uses <ngx-control-wrapper> for automatic ARIA wiring, error display, and warning support.
  • No hidden proxies — each [ngModel] binding registers its own field path directly.
  • No manual error aggregation — the library handles error/warning display per field.
  • Bidirectional validation via validationConfig still cross-validates both dates.
  • Trade-off: fields appear as separate controls instead of a unified date-range widget.
} @else {
  • Hidden proxy fields register departureDate and returnDate in the form tree.
  • Adapter component provides a single composite date range UI.
  • Fan-out via setValueAtPath writes the adapter value back to separate model fields.
  • Error aggregation merges errors from both fields into the adapter display.
  • Display mode gating — the adapter tracks touched/submitted state to match on-blur-or-submit behavior.
  • Bidirectional validation via validationConfig ensures each date change revalidates the other.
}
